Why use it

🛡️

Pre-flight validation

Detects duplicate MACs, trunked vNICs, and multi-VLAN VMs before touching anything.

🔁

Rolling & retried

One subnet at a time, with a single automatic retry on first failure — just like the runbook.

📡

Live port polling

Background TCP-2121 probing of CVMs/hosts for connectivity observability during migration.

✅

Post-verify

Confirms Advanced state, NIC connectivity, and drives the task to SUCCEEDED.

🎛️

Interactive TUI

Arrow-key menus, colored tables, checkbox subnet selection, and a live summary.

📦

Zero footprint

Everything lives in a project-local .venv. Delete the folder to uninstall.

How it works

Validate (read-only)

Enumerates Basic VLAN subnets and their VMs/NICs via networking & vmm, and reports every finding before any change.

Migrate

Calls the public migrate-subnets action (the supported equivalent of the internal kVlanSubnetMigration RPC) while polling port 2121 in the background.

Verify

Tracks the task through the prism Tasks API to SUCCEEDED, then confirms the subnet is Advanced and NICs are still connected.

Usage

Launch the interactive UI, or drive it non-interactively for automation:

# Interactive menu (global command, run from anywhere)
vlan-migrator

# Non-interactive
vlan-migrator list-basic                          # list Basic VLAN subnets
vlan-migrator validate --all                      # validate everything (no changes)
vlan-migrator migrate --subnet vlan100 --dry-run  # show the plan
vlan-migrator migrate --all                       # rolling migrate all eligible

Requires Prism Central 7.3+/AOS 7.3+ with the Network Controller enabled. See the full README for configuration and safety notes.
